C7-1 (HIGH) — `?kind=` query param is dead + locale mismatch
------------------------------------------------------------

FabBottomSheet emits `/app/transactions/neu?kind=ausgabe|einnahme|spende`,
but:
  - `+page.server.ts` never reads `url.searchParams.get('kind')`
  - `+page.svelte:10` hardcodes `selectedType = 'expense'`
  - Locale mismatch: German `ausgabe` vs domain enum `expense`

Failing tests (RED):
  - tests/unit/transaction-kind-url.test.ts — parseKindFromUrl maps DE↔EN
    consistently (and returns null on bogus input)
  - src/routes/app/transactions/neu/page.server.test.ts — `load()` honours
    `?kind=einnahme` and surfaces `data.initialType === 'income'`
  - E2E (c7-mobile-iphone12.spec.ts): after tapping "Neue Einnahme" the form
    type-tabs reflect aria-pressed="true" on the Einnahme tab.

Fix: NEW src/lib/domain/transaction-kind-url.ts with KIND_DE_TO_EN map and
parseKindFromUrl(); read in load(); pass `initialType` to +page.svelte; wire
into $state initial value.

C7-2 (HIGH) — E2E spec non-loadable
------------------------------------

`test.use({...devices['iPhone 12']})` inside `test.describe` is rejected by
Playwright (forces a new worker). Spec file fails to LOAD; PM-003 never
runs.

Failing check (RED):
  - `npx playwright test --list tests/e2e/c7-mobile-polish.spec.ts` errors
    out with "Cannot use({ defaultBrowserType }) in a describe group".

Fix: split into one file per device emulation (c7-mobile-iphone12.spec.ts,
c7-mobile-pixel5.spec.ts, c7-tablet-ipad.spec.ts) — each has top-level
`test.use(devices[...])`. Replace the original c7-mobile-polish.spec.ts.

C7-3 (MEDIUM) — Topbar wordmark visible on mobile (PM-010)
----------------------------------------------------------

Topbar.svelte has `<span class="flex-1 text-sm font-semibold ... sm:hidden">
Folge der Wolke</span>` — this is shown ON mobile (< sm) and hidden ON
sm+. The PM-010 requirement is the OPPOSITE: hide the wordmark on small
screens so the topbar gets back the horizontal space.

Failing test (RED):
  - src/lib/components/admin/Topbar.test.ts — checks the wordmark span has
    `hidden` or breakpoint utility hiding it BELOW sm/md.

Fix: switch class to `hidden sm:hidden md:flex` (or simply remove on
mobile). Pair with .safe-top utility.
